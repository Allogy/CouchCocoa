Detailed Description

Represents an HTTP request to a RESTResource, and its response.

Can be used either synchronously or asynchronously. Methods that return information about the response, such as -httpStatus or -body, will block if called before the response is available. Or you can explicitly block by calling -wait. On the other hand, to avoid blocking you can call -onCompletion: to schedule an Objective-C block to run when the response is complete. (Yes, the non-blocking mode takes a block... :)

- (BOOL) wait

Blocks till any pending network operation finishes (i.e.

-isComplete becomes true.) -start will be called if it hasn't yet been. On completion, any pending onCompletion blocks are called first, before this method returns. The synchronous methods below all end up calling this one.

Returns:
YES on success, NO on error.
- (BOOL) wait: (NSError **)  outError

Same as -wait but also returns any resulting error in the outError parameter.

This is useful if the receiver is an intermediate value not accessible in a variable, for instance if you do something like [[resource GET] wait: &error]

The HTTP request body.

Cannot be changed after the operation starts.

- (NSError*) error [read, retain]

If the request has failed, this will be set to an NSError describing what went wrong; else it's nil.

An HTTP response status of 300 or greater is considered an error and will cause this property to be set. This method does not block, but it won't be set to a non-nil value until the operation finishes.

- (int) httpStatus [read, assign]

HTTP status code of the response (Synchronous.) Until the request finishes, this is zero.

It's also zero if a lower-level network error occurred (like if the host couldn't be found or the TCP connection was reset.)

- (NSHTTPURLResponse*) response [read, assign]

The raw NSHTTPURLResponse object, in case you need it.

- (id) resultObject [read, write, retain]

Object associated with this response.

A client can store anything it wants here, typically a value parsed from or represented by the response body; often this property will be set by an onCompletion block.
