Detailed Description

A model object representing a persistent replication to or from another database.

Each instance represents a document in the server's special _replication database. Instances are created by the -replicate... factory methods on CouchDatabase.

Member Function Documentation

- (void) actAsUser: (NSString *)  username
withRoles: (NSArray *)  roles 

Sets the "user_ctx" property of the replication, which identifies what privileges it will run with when accessing the local server.

To replicate design documents, this should be set to a value with "_admin" in the list of roles. The server will not let you specify privileges you don't have, so the request to create the replication must be made with credentials that match what you're setting here, unless the server is in no-authentication "admin party" mode. See <https://gist.github.com/832610>, section 8, for details. If both 'user' and 'roles' are nil, the user_ctx will be cleared.

Parameters:
usernameA server username, or nil
rolesAn array of CouchDB role name strings, or nil
