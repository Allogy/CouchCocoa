Detailed Description

Manages an embedded instance of CouchDB that runs in a background thread.

Property Documentation

- (id<CouchbaseDelegate>) delegate [read, write, assign]

The delegate object, which will be notified when the server starts.

- (NSURL*) serverURL [read, retain]

The HTTP URL the server is listening on.

Will be nil until the server has finished starting up, some time after -start is called. This property is KV-observable, so an alternative to setting a delegate is to observe this property and the -error property and wait for one of them to become non-nil.

- (NSError*) error [read, retain]

If the server fails to start up, this will be set to a description of the error.

This is KV-observable.

- (NSURLCredential*) adminCredential [read, assign]

A credential containing the admin username and password of the server.

These are required in any requests sent to the server. The password is generated randomly on first launch.

- (NSString*) rootDirectory [read, write, copy]

The root directory where Couchbase Mobile will store data files.

This defaults to ~/CouchbaseMobile. You may NOT change this after starting the server.

- (NSString*) logDirectory [read, assign]

The directory where CouchDB writes its log files.

- (NSString*) databaseDirectory [read, assign]

The directory where CouchDB stores its database files.

- (NSString*) iniFilePath [read, write, copy]

The path to an app-specific CouchDB configuration (".ini") file.

Optional; defaults to nil. The settings in this file will override the default CouchDB settings in default.ini, but will in turn be overridden by any locally-made settings (see -localIniFilePath).

- (NSString*) localIniFilePath [read, assign]

The path to the mutable local configuration file.

This starts out empty, but will be modified if the app sends PUT requests to the server's _config URI. The app can restore the default configuration at launch by deleting or emptying the file at this path before calling -start.

- (uint8_t) logLevel [read, write, assign]

Controls the amount of logging by Erlang and CouchDB.

Defaults to 0, meaning none. 1 logs errors only, 2 also logs CouchDB info (like HTTP requests), 3 logs Erlang 'progress'.
